Táblákban alkalmazható korlátozások

PRIMARY KEY

A PRIMARY KEY egyedi azonosítót biztosít egy tábla minden sorának. Egy táblában csak egy PRIMARY KEY lehet.

CREATE TABLE Diak (
    ID INT PRIMARY KEY,
    Nev VARCHAR(100)
);

FOREIGN KEY

A FOREIGN KEY egy másik tábla PRIMARY KEY oszlopára hivatkozik, biztosítva az adatintegritást.

CREATE TABLE Beiratkozas (
    DiakID INT,
    TantargyID INT,
    FOREIGN KEY (DiakID) REFERENCES Diak(ID)
);

UNIQUE

Az UNIQUE biztosítja, hogy egy oszlop értékei egyediek legyenek.

CREATE TABLE Felhasznalo (
    ID INT PRIMARY KEY,
    Email VARCHAR(255) UNIQUE
);

NOT NULL

A NOT NULL megakadályozza, hogy egy oszlop értéke NULL legyen.

CREATE TABLE Termek (
    ID INT PRIMARY KEY,
    Nev VARCHAR(100) NOT NULL
);

DEFAULT

A DEFAULT egy alapértelmezett értéket ad egy oszlopnak, ha nincs megadva érték.

CREATE TABLE Felhasznalo (
    ID INT PRIMARY KEY,
    RegisztracioDatuma DATE DEFAULT CURRENT_DATE
);

CHECK

A CHECK egy feltételt állít fel, amelynek az oszlop értékeinek meg kell felelniük.

CREATE TABLE Dolgozo (
    ID INT PRIMARY KEY,
    Fizetes INT CHECK (Fizetes > 0)
);
